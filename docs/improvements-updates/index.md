This section documents product, UX, and implementation decisions that have shaped the Household Budget Planner over time. Each entry is a focused decision note covering what changed, why it changed, and how it affects the product.

Use this section to understand the reasoning behind specific behavior, review product decisions as a team, and track the evolution of the application.

## How to Add a New Update Note

1. Copy the [update note template](./update-template.html) into a new Markdown file in this folder
2. Name the file clearly and descriptively (for example, `expense-recurring-support.md`)
3. Fill in all sections, removing any that do not apply
4. Add a row to the table above with a brief summary
5. Link back to affected core docs (architecture, API reference, frontend guide, roadmap) where relevant
